# 社会人仿真技能说明 / Social Human Simulation Skill Guide

这个仓库目前采用极简 ClaudeSkill 风格：每个技能都是一个 `skills/<name>/SKILL.md`，frontmatter 尽量保持简单。技能内部通过正文说明应该读取哪些 workspace 状态、如何推理、写出哪些 `state/*.json`。

This repository follows a minimal ClaudeSkill-style layout: each skill lives in `skills/<name>/SKILL.md`, with optional deterministic scripts and research notes. The static docs are intended to be readable by both Chinese and English users.

## 理论依据在哪里 / Where To Find The Theory

- 全局理论索引 / Global research index: `theory_basis.md`
- 每个技能页面 / Each generated skill page: `docs/skills/<skill>.md`
- 源文件 / Source reference files: `skills/<skill>/references/*.md`
- 代码 baseline / Deterministic scripts: `skills/<skill>/scripts/*.py`

如果技能使用了公式、曲线或社会科学模型，理论依据会放在 `references/` 中，并由 `scripts/generate_skill_catalog.py` 自动渲染到静态技能页面。

If a skill uses a curve, formula, or social-science model, the rationale lives in `references/` and is automatically rendered into the static skill page.

## 总体运行逻辑

一个社会人的行为不是由单个技能决定，而是由多个技能写出的状态共同影响。推荐先用确定性脚本更新可解释数值，再让 LLM 做情境化解释和少量修正：

```text
observation
  -> circadian / physiology / routine / economy / culture / relationships / norms / health / affordance
  -> learning / moral_judgment / media_literacy / civic_institution / reflection
  -> cognition(appraisal + TPB)
  -> plan
  -> memory(Ebbinghaus + ACT-R activation)
```

其中：

- `observation` 负责看见世界。
- 领域技能负责更新人的某个侧面，比如身体、经济、文化、社会关系。
- `cognition` 把这些状态综合成情绪和意图。
- `plan` 把意图变成行动。
- `memory` 把重要经历留下来，供后续技能读取。

English summary:

- `observation` grounds the agent in the current world.
- Domain skills update body, economy, culture, relationships, norms, institutions, media exposure, and learning.
- `cognition` turns these states into emotion and intention.
- `plan` turns intention into action.
- `memory` preserves important events and lets low-value memories fade.

## 当前技能

### observation

作用：获取当前 tick 的环境感知。

内部逻辑：

- 调用环境的 `<observe>`。
- 把自然语言观察写到 `state/observation.txt`。
- 如果环境返回结构化上下文，就写到 `state/observation_ctx.json`。

它是其他技能的事实基础。没有观察，其他技能只能靠旧状态和 profile 推断。

### circadian

作用：模拟昼夜节律、睡眠压力、清醒度、食欲节律和 chronotype。

内部逻辑：

- 从观察里读取当前时间。
- 更新 24 小时节律相位。
- 用两过程睡眠模型思路更新：
  - 清醒时间越长，`sleep_pressure` 越高。
  - 昼夜节律决定某个时间段更清醒还是更困。
- 计算 `circadian_alertness`、`circadian_appetite`、`sleep_tendency`。

输出：

- `state/circadian.json`
- `state/sleep.json`

它会影响 `physiology` 对饥饿、疲劳的计算，也会影响 `cognition` 的行动控制感。

### physiology

作用：模拟身体层面的需求，如饥饿、饱腹、口渴、疲劳、压力、疼痛、疾病。

内部逻辑：

- 读取观察、旧的身体状态、昼夜节律、健康状态。
- 根据时间、活动、进食、饮水、睡眠、压力事件更新身体变量。
- 饥饿不是简单数值，而是一个压力：

```text
hunger_pressure =
  距离上次进食的影响
  + 昼夜食欲影响
  + 活动消耗
  + 压力对食欲的影响
  - 饱腹感
```

输出：

- `state/physiology.json`
- `state/needs.json`
- 必要时追加 `state/body_events.jsonl`

它给 `cognition` 提供“现在最急的身体需求”，也给 `plan` 提供是否应该中断当前计划的依据。

### routine

作用：维护日常作息、习惯、时间段期待和重复生活模式。

内部逻辑：

- 根据当前时间、地点、角色、身体状态判断“此时通常该做什么”。
- 维护习惯的 cue、action、reward、strength。
- 重复行为会增强习惯，长期不做会弱化。

输出：

- `state/routine.json`
- `state/habits.json`

它让 agent 不会每个 tick 都像第一次生活，而是有稳定的日常节奏。

### economy

作用：模拟经济生活，包括现金、收入、支出、工作义务、消费选择、稀缺压力和满意化选择。

内部逻辑：

- 更新现金、库存、收入预期、固定支出、债务和工作安排。
- 判断行动是否买得起、是否太贵、是否会造成预算压力。
- 将“经济紧张”转化成 `scarcity_pressure` 和 `budget_stress`。
- 在时间、信息或注意力有限时，不假设 agent 会全局最优化；只要某个方案达到动态 `aspiration_level`，就可以停止搜索并选择它。

输出：

- `state/economy.json`
- `state/resources.json`

它会影响吃什么、怎么通勤、是否消费、是否按时上班，也会影响压力、情绪和搜索/比较的耐心。

### culture

作用：模拟文化背景、礼仪、禁忌、象征意义、节日和群体价值。

内部逻辑：

- 识别当前文化场景，比如家庭聚餐、职场会议、公共空间、节日。
- 判断哪些行为礼貌、冒犯、体面、丢脸、亲密或禁忌。
- 估计文化舒适度和误解风险。

输出：

- `state/culture.json`

它不应该刻板化人物，而是基于 profile、场景、关系和观察来推断。

### relationships

作用：维护人与人之间的熟悉度、信任、喜欢、义务、冲突和共同历史。

内部逻辑：

- 从互动事件中识别对象和事件性质。
- 更新每个关系对象的 `familiarity`、`trust`、`liking`、`obligation`、`conflict`。
- 信任和冲突变化较慢，强事件才会大幅改变。
- 如果对方提供意见或建议，按信任、尊重、熟悉度、专业线索和冲突更新 `influence_weight`，供后续共识/社会影响判断使用。

输出：

- `state/relationships.json`
- 必要时追加 `state/social_events.jsonl`

它让同一个人下次再出现时，不再是陌生 NPC；如果多个可信对象反复表达相近意见，也能形成 DeGroot-style 的社会影响信号。

### norms

作用：模拟社会规范、角色义务、场所规则、许可、违规风险和制裁。

内部逻辑：

- 识别当前角色：顾客、员工、朋友、陌生人、病人、学生等。
- 识别当前地点的规则：排队、安静、守时、不能插队、不能冒犯他人。
- 估计 `norm_pressure`、`violation_risk`、`witness_visibility`。

输出：

- `state/norms.json`

它会影响 `cognition` 里的主观规范压力，也会让违规行为带来羞耻、内疚、声誉或制度后果。

### health

作用：模拟健康状态、疼痛、疾病、慢性压力、恢复能力、运动和长期福祉。

内部逻辑：

- 区分普通疲劳、疾病、疼痛、长期压力。
- 慢变量缓慢变化，急性症状由事件触发。
- 计算健康对移动、食欲、社交意愿和行动控制感的影响。

输出：

- `state/health.json`

它和 `physiology` 相互影响，但更偏长期健康和症状，不只是当前饥饿或困倦。

### affordance

作用：判断当前环境和自身状态下，哪些行动真实可行、昂贵、有风险或被阻止。

内部逻辑：

- 读取环境可用动作。
- 综合距离、时间、开放时间、钱、访问权限、天气、疲劳、疼痛、规范、拥挤等约束。
- 把候选行动分成 feasible、costly、risky、blocked、unknown。

输出：

- `state/affordances.json`

它不决定 agent 想做什么，只告诉 `plan` 什么做得到、代价是什么。

### reflection

作用：把记忆压缩成更稳定的信念、偏好、自我认知和未来倾向。

内部逻辑：

- 周期性读取记忆、关系、情绪、身体和计划结果。
- 找重复模式，比如“我跳过午饭后容易烦躁”。
- 只有证据足够时才更新信念或偏好。

输出：

- `state/reflections.jsonl`
- `state/beliefs.json`
- `state/preferences.json`
- `state/self_concept.json`

它比 `memory` 慢，不是每 tick 都跑。`memory` 记录事实，`reflection` 总结模式。

### learning

作用：维护技能学习、知识保持、练习次数、自动化程度、自我效能、动机质量和复习间隔。

内部逻辑：

- 从观察、计划结果、情绪和旧学习状态中识别练习、成功、失败、示范、鼓励、批评。
- 熟练度采用递减收益：越接近熟练，单次练习提升越小。
- 保持率随时间衰减。
- 复习间隔服务于最终保持目标：近期要用的信息更快复习，长期要用的信息用更长间隔；刚成功回忆后立刻重复收益较低。
- 自动化程度随稳定情境中的重复行为上升。
- 自我效能根据 Bandura 的四类来源更新：直接成功、观察他人、社会劝说、情绪/生理状态。
- 动机质量根据 Self-Determination Theory 更新：自主、胜任、关系和内化越高，越可能主动、持续地学习；被迫、失败和孤立会提高 `amotivation_risk`。
- 可选脚本：`skills/learning/scripts/update_learning.py`

输出：

- `state/learning.json`
- 必要时追加 `state/learning_events.jsonl`

它让 agent 不只是记住事实，还会逐渐“会做事”、对某些任务更有或更没有信心，并能区分被迫应付和真正内化的学习。

### moral_judgment

作用：模拟道德评价、道德情绪和修复/惩罚/帮助倾向。

内部逻辑：

- 用 moral foundations 估计关怀/伤害、公平/欺骗、忠诚/背叛、权威/颠覆、圣洁/降格、自由/压迫。
- 区分快速道德直觉与需要进一步审慎推理的情况。
- 计算内疚、羞耻、愤怒、同情、厌恶、钦佩等道德情绪。
- 可选脚本：`skills/moral_judgment/scripts/update_moral_judgment.py`

输出：

- `state/moral_appraisal.json`
- 必要时追加 `state/moral_events.jsonl`

它会影响规范压力、关系信任/冲突、道歉、补偿、回避、惩罚和帮助行为。

### media_literacy

作用：模拟 agent 面对新闻、谣言、广告、社交媒体、朋友转述和机构声明时如何更新信念。

内部逻辑：

- 区分来源可信度、证据质量、重复带来的熟悉感、身份一致性、情绪唤起、误导风险和预防接种强度。
- 重复只增加熟悉感，不等于证据质量。
- 预辟谣/事实核查会提高后续抵抗类似操纵话术的能力。
- 可选脚本：`skills/media_literacy/scripts/update_media_literacy.py`

输出：

- `state/media_literacy.json`
- `state/belief_update_hints.json`

它让不同 agent 即使看到同一个说法，也会因为信任、身份、证据和情绪不同而产生不同接受倾向。

### civic_institution

作用：模拟人与制度互动，包括政府、学校、医院、警察、公司、房东、银行等。

内部逻辑：

- 识别活跃制度和角色：病人、学生、员工、租户、公民、申请者等。
- 用程序正义估计 voice、neutrality、respect、trustworthy motives。
- 估计费用、材料、等待、歧视风险、交通/健康限制等进入门槛。
- 更新制度信任、合法性感、合规压力和服务结果。
- 可选脚本：`skills/civic_institution/scripts/update_civic_institution.py`

输出：

- `state/institutions.json`
- 必要时追加 `state/institution_events.jsonl`

它让“去医院/报警/上班/办证/交租”不只是地点动作，而是带有等待、规则、权利、义务、信任和压力的制度事件。

### cognition

作用：把观察、需求、记忆、身体、经济、文化、关系、规范等状态综合成情绪和意图。

内部逻辑：

- 用 Scherer-style appraisal 估计新奇性、愉悦性、目标促进/阻碍、紧急性、可控性、规范压力。
- 维持短期情绪、长期 mood、人格影响，并限制单 tick 情绪跳变。
- 根据 Theory of Planned Behavior 计算意图：
  - `attitude`
  - `subjective_norm`
  - `perceived_control`
- 在不确定和认知负荷高时，显式记录代表性、可得性、锚定调整等启发式如何影响判断。
- 对低风险、时间有限或 routine 选择使用满意化搜索：不是找全局最优，而是找“足够好且可行”的方案。
- 情绪会修正意图选择，例如恐惧降低冒险意愿，疲劳降低控制感。
- 可选脚本：`skills/cognition/scripts/update_cognition.py`

输出：

- `state/emotion.json`
- `state/intention.json`

它是“想做什么”的核心，但它应该读取其他技能产出的状态，而不是自己包办所有领域逻辑。启发式和满意化选择应作为可解释的 bias/process 字段写出，而不是伪装成客观事实。

### plan

作用：把当前意图变成环境行动。

内部逻辑：

- 简单熟悉行为走 System 1：直接行动。
- 复杂陌生行为走 System 2：生成多步计划。
- 遇到身体急需、环境失败、规范风险或约束不可行时中断或调整计划。

输出：

- `state/plan_state.json`

它是执行层，应该参考 `state/affordances.json`、`state/needs.json`、`state/norms.json` 等约束。

### memory

作用：记录重要经历，并通过遗忘曲线、检索激活和间隔复习模拟长期记忆衰减。

内部逻辑：

- 决定当前 tick 是否值得记忆。
- 重要事件写入 JSONL。
- 单次记忆使用 Ebbinghaus-style 指数衰减。
- 反复遇到或反复回忆的记忆使用 ACT-R base-level activation 提升检索概率。
- 重要记忆可以带 `_target_retention_interval`、`_spacing_ratio`、`_next_review_tick` 等字段，让记忆维护既能遗忘，也能按目标重新激活。
- 记忆有重要性、标签、保留度、激活值和检索概率。

输出：

- `state/memory.jsonl`

它是所有长期连续性的基础，`relationships`、`reflection`、`cognition` 都会依赖它。这个设计让“只发生一次的小事会淡出”，但“经常见到的人、地点、承诺和规则会稳定保留”，并且可以根据未来使用目标安排复习。

## 后续建议

下一批可以继续补：

- `communication`：更细的对话轮次、面子、含蓄表达、修复话语。
- `identity`：身份认同、阶层、职业、家庭角色、价值观冲突。
- `politics`：公共议题立场、集体行动、投票/抗议/组织参与。
- `attention`：注意力、分心、信息过载、手机使用。
